# PRD - sisiduck 티어메이커 (v1)

## 1. 개요

### 제품 비전
sisiduck(시시덕)은 재미있는 콘텐츠를 모아놓는 플랫폼이다. 첫 번째 기능으로 **티어메이커**를 제공한다. 사용자가 다양한 주제(만화 캐릭터, 게임 챔피언, 음식 등)에 대해 S/A/B/C/D/F 등급으로 아이템을 배치하고, 다른 사람들의 티어와 비교할 수 있다.

### 핵심 차별점
- **공유 아이템 풀**: 소분류(예: 원피스) 단위로 아이템 이미지를 공유. 누구나 추가/교체 가능한 위키 방식.
- **토픽 기반 구조**: 같은 소재에서 다양한 관점의 토픽을 만들 수 있음 (밀짚모자 전투력, 악마의 열매 랭킹 등).
- **플랫폼 확장성**: 향후 이상형 월드컵, 퀴즈, 미니게임 등 추가 가능한 구조.

### 타겟 사용자
- 한국 사용자 (한국어 우선)
- 만화, 게임, 영화 등 콘텐츠 소비자
- 티어 리스트를 만들고 공유하며 즐기고 싶은 사용자

### SEO 전략
- 도메인: `sisiduck.com`
- SEO 타이틀: `티어표 만들기, 티어 랭킹 - 시시덕`
- 브랜드와 검색 키워드 분리 전략 (피쿠/이상형월드컵 방식)

## 2. 콘텐츠 계층 구조

```
대분류 카테고리 (관리자 고정)
└── 소분류 카테고리 (사용자 생성, 별칭 + 정규화 중복 방지)
    ├── 📦 공유 아이템 풀 (소분류에 귀속, 누구나 추가/이미지 교체)
    └── 토픽 (사용자 생성, 풀에서 아이템 선택)
        └── 티어리스트 (개인 생성, 드래그앤드롭 배치)
```

## 3. 기능 요구사항

### 3.1 인증

| ID | 기능 | 설명 |
|----|------|------|
| AUTH-01 | 소셜 로그인 | Google, GitHub OAuth를 통한 로그인 |
| AUTH-02 | 이메일/비밀번호 | 이메일 기반 회원가입/로그인 |
| AUTH-03 | 비로그인 체험 | 티어 만들기 체험 가능, 저장 시 로그인 유도 |

### 3.2 카테고리 & 소분류

| ID | 기능 | 설명 |
|----|------|------|
| CAT-01 | 대분류 관리 | 관리자가 고정 카테고리 관리 (만화/애니, 게임, 영화, 음식, 스포츠, 음악 등) |
| CAT-02 | 소분류 생성 | 로그인 사용자가 토픽 생성 시 함께 생성 가능 |
| CAT-03 | 소분류 중복 방지 | 정규화 매칭 (공백/대소문자/특수문자 무시), 유사 소분류 존재 시 추천 |
| CAT-04 | 별칭 시스템 | 소분류에 여러 이름 등록 가능 (원피스 = ONE PIECE = ワンピース) |
| CAT-05 | 사이드바 표시 | 대분류 > 소분류 아코디언으로 항상 노출 |

### 3.3 공유 아이템 풀

| ID | 기능 | 설명 |
|----|------|------|
| ITEM-01 | 아이템 추가 | 로그인 사용자가 소분류에 아이템(이름 + 이미지) 추가 |
| ITEM-02 | 이미지 교체 | 기존 아이템의 이미지를 더 좋은 이미지로 교체 가능 |
| ITEM-03 | 변경 히스토리 | 이미지 변경 이력 보존 |
| ITEM-04 | 토픽 내 추가 | 토픽 생성/편집 중 풀에 없는 아이템을 바로 추가 (풀에도 자동 반영) |

### 3.4 토픽

| ID | 기능 | 설명 |
|----|------|------|
| TOPIC-01 | 토픽 생성 | 대분류 선택 → 소분류 선택/생성 → 제목 입력 → 아이템 풀에서 선택 |
| TOPIC-02 | 토픽 상세 | 선택된 아이템 목록 + 티어리스트 피드 표시 |
| TOPIC-03 | 티어리스트 피드 | 해당 토픽의 티어리스트들 나열. 인기순(기본)/최신순 정렬 |

### 3.5 티어 에디터 (핵심)

| ID | 기능 | 설명 |
|----|------|------|
| TIER-01 | 드래그 앤 드롭 | 미배치 영역의 아이템을 S/A/B/C/D/F 행으로 드래그하여 배치 |
| TIER-02 | 티어 행 커스텀 | 행 추가/삭제, 이름 변경, 색상 변경 |
| TIER-03 | 저장 | 로그인 사용자만 저장 가능. 비로그인 시 로그인 유도 팝업 |
| TIER-04 | 이미지 내보내기 | 완성된 티어를 PNG 이미지로 다운로드 |

### 3.6 티어리스트 상세 & 소셜

| ID | 기능 | 설명 |
|----|------|------|
| SOCIAL-01 | 티어리스트 보기 | 다른 사용자의 완성 티어 확인 |
| SOCIAL-02 | 좋아요 | 티어리스트에 좋아요 |
| SOCIAL-03 | 댓글 | 티어리스트에 댓글 작성 |
| SOCIAL-04 | 공유 | URL 복사 + 이미지 다운로드 |

## 6. 데이터 모델

### 테이블 관계

```
users
├── tier_lists (1:N)
├── topics (1:N)
├── items (1:N, 기여)
├── likes (1:N)
└── comments (1:N)

categories (1:N) → subcategories
subcategories
├── subcategory_aliases (1:N)
├── items (1:N, 공유 아이템 풀)
│   └── item_image_history (1:N)
└── topics (1:N)
    ├── topic_items (M:N, items 참조)
    └── tier_lists (1:N)
        ├── tier_rankings (1:N, items 참조)
        ├── likes (1:N)
        └── comments (1:N)
```

### 테이블 상세

| 테이블 | 주요 컬럼 | 설명 |
|--------|----------|------|
| `users` | id, email, nickname, avatar_url, provider | 사용자 |
| `categories` | id, name, slug, icon, sort_order | 대분류 (관리자 고정) |
| `subcategories` | id, category_id, name, slug, normalized_name, created_by | 소분류 |
| `subcategory_aliases` | id, subcategory_id, alias, normalized_alias | 소분류 별칭 |
| `items` | id, subcategory_id, name, image_url, created_by | 공유 아이템 |
| `item_image_history` | id, item_id, image_url, changed_by, changed_at | 이미지 변경 이력 |
| `topics` | id, subcategory_id, title, slug, description, created_by | 토픽 |
| `topic_items` | topic_id, item_id | 토픽-아이템 연결 |
| `tier_lists` | id, topic_id, user_id, created_at | 티어리스트 |
| `tier_rankings` | id, tier_list_id, item_id, tier, position | 티어 배치 |
| `likes` | user_id, tier_list_id | 좋아요 |
| `comments` | id, tier_list_id, user_id, content, created_at | 댓글 |

## 8. 개발 우선순위 (마일스톤)

### Phase 1: 핵심 기능 (MVP)
- 프로젝트 세팅 (Next.js + Supabase + Tailwind)
- 인증 (소셜 로그인)
- 대분류/소분류 카테고리 + 사이드바
- 공유 아이템 풀 (추가/편집)
- 토픽 CRUD
- 티어 에디터 (드래그앤드롭 + 저장)
- 티어리스트 상세 보기

### Phase 2: 소셜 & 탐색
- 좋아요/댓글
- 마이페이지
- 통합 검색
- 티어리스트 피드 (인기순/최신순)
- 이미지 내보내기

### Phase 3: 고도화
- 소분류 별칭 시스템
- 아이템 이미지 히스토리
- OG Image 자동 생성
- 반응형/모바일 최적화
- 플랫폼 홈 랜딩 페이지

### Phase 4: 플랫폼 확장 (향후)
- 이상형 월드컵
- 퀴즈/미니게임
- 기타 재미 콘텐츠
